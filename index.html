jspm.io

Frictionless browser package management

_jspm is a browser module loader that will load any module format (ES6 / AMD / CJS) locally or even directly from NPM and Github utilising SPDY push._ #### JSPM Loader * Write ES6 modules today and use them in any browser. * Load AMD and CommonJS modules together in the same application. * Load global scripts with a sandboxed global object. * Built on top of the [ES6 Module Loader polyfill](https://github.com/ModuleLoader/es6-module-loader), making it standards-compliant and future-friendly. #### JSPM CDN * Load an NPM module in the browser without any build or install. * Load dependency-managed modules directly from Github. * Worried about perfomance? SPDY push is used to automatically bundle dependencies together, minimising round trips. ### Examples **Write and load an ES6 module** es6-module.js: ```javascript import { someProperty } from './some-dependency'; export var hello = 'es6'; ``` main.js: ```javascript jspm.import('es6-module', function(m) { console.log(m.hello); }); ``` Any local modules, including AMD and CommonJS can be loaded in this way. RequireJS-style configuration offers loader flexibility. **Import the latest revision of a module directly from NPM** ```javascript jspm.import('npm:underscore@1.5', function(underscore) { var _ = underscore.default; }); ``` The module is loaded through the CDN along with its version-managed dependencies. **Load a file from the latest stable version tag of a library on Github** ```javascript <link rel="stylesheet" href="https://github.jspm.io/hakimel/reveal.js/css/reveal.css"></link> ``` The CDN can be used on its own to load any script, stylesheets or HTML import resources directly from NPM or Github. SPDY push still manages dependencies minimising round trips, making this an efficient CDN option. **Import the latest stable version of a module from NPM ([see this example on jsbin](http://jsbin.com/oqatol/1/edit))** ```javascript jspm.import('npm:spinning', function(m) { var Spinning = m.default; var spinner = Spinning().text('loading...').light().size(200); setTimeout(function () { spinner.remove(); }, 3000); }); ``` **Load an AMD module from the latest stable version of a library on Github** ```javascript jspm.import('github:mout/mout/src/array/map', function(m) { var map = m.default; }); ``` #### Getting Started Include the jspm loader in the head of the page: ```html <script src="https://jspm.io/loader.js"></script> ``` Alternatively [download the loader and include it locally](https://github.com/jspm/jspm-loader). To require a local module, set the baseURL and load from it (by default the baseURL is set to the current page location): ```javascript jspm.config({ baseURL: 'http://www.mysite.com/lib' }); jspm.import('local/file', function(module) { }); ``` To require from Github or NPM, use the location name in the require: ```javascript jspm.import([ 'github:username/repo@3.4/main.js', 'npm:some-module@1.2' ], function(m1, m2) { }); ``` Alternatively, just start writing scripts in your favourite module format: main.js: ```javascript var githubModule = require('github:username/repo@3.4/main.js'); var npmModule = require('npm:some-module@1.2'); module.exports = { hello: 'world' }; ``` ```javascript jspm.import('main', function(m) { console.log(m.default.hello); // world }); ``` The module format is detected automatically and handled appropriately. AMD, CommonJS, ES6 and global scripts are all supported. The loader supports RequireJS-style configuration options (map & shim configuration) as well as a plugin system. [Read more at the github page](https://github.com/jspm/jspm-loader). #### CDN Endpoints The CDN endpoints are entirely URL-based. The basic format is: ``` https://npm.jspm.io/[module name]@[version]/[file path] https://github.jspm.io/[username]/[repo]@[version]/[file path] ``` Typically a minor version is specified only (eg `@2.2`), which will load the latest revision. This is the recommended way of loading a resource as it allows patches but not breaking changes in the dependency tree. If no version is specified, the latest stable version is loaded. Otherwise a complete version or tagname can also be provided. _The CDN endpoints don't need to be used with the jspm loader, they can also be used with scripts, stylesheets or HTML imports in the page._ The benefits of having SPDY push dependencies mean that imported resources in styles, scripts and HTML imports don't require a separate round trip. Script dependencies are traced automatically and provided with this support. #### Feedback & Roadmap _Note that it is still early days and this work is still experimental._ If you are interested in being notified on future jspm announcements, sign up to stay informed here: <form class="signup-group"> <input type="text" class="signup" value="Notify me of future announcements"></input> <input type="submit" class="signup-submit" value="Sign up"></input> <div class="cog"></div> </form> If you find any bugs or want to request a feature, [use the issue tracker here](https://github.com/jspm/issues/issues).
