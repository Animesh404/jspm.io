<div class="loader-icon"></div> #### jspm Loader * RequireJS-style module loader with similar configuration and plugin system. * Built on top of the [ES6 Module Loader polyfill](https://github.com/ModuleLoader/es6-module-loader), making it standards-compliant and future-friendly. * Write ES6 modules today and use them in any browser. * Load AMD and CommonJS modules together in the same application. * Load global scripts with a dependency-managed global object. [Read the loader documentation on Github](https://github.com/jspm/jspm-loader) <div class="cdn-icon"></div> #### CDN Endpoints & Registry * Custom endpoints allow loading modular code without any build or install. * By default, the loader comes with the `github` and `npm` endpoints, provided as a free service. Anyone can create an endpoint or endpoint mirror. * Load an npm module in the browser without any build or install. * Load dependency-managed modules directly from Github. * Dependencies traced and served with SPDY push, minimising download time. * The registry endpoint allows shortcut names such as `jquery`. _The CDN endpoint code will be open-sourced, allowing anyone to create an endpoint mirror or service._ [View the registry page on Github](https://github.com/jspm/registry) <div class="cli-icon"></div> #### jspm CLI * Download modules and dependencies from the jspm registry, npm or Github, with the jspm loader map configuration created automatically. * Endpoints provide their own download implementations making the CLI truly modular. * Because the CLI understands the loader, modular code works without the need for complex manual configuration. * Build ES6 into ES5, transpile ES6 modules into AMD and uglify with source maps support. [Read the CLI documentation on Github](https://github.com/jspm/jspm-cli) ### Examples **Include the jspm loader in a page** ```html <script src="https://jspm.io/loader.js"></script> ``` **Import the latest revision of a module directly from npm** ```html <script> jspm.import('npm:underscore@1.5', function(_) { _.each([1, 2, 3], console.log); }); </script> ``` Loads from the npm endpoint URL, `https://npm.jspm.io/underscore@1.5.js`. The module is loaded through the CDN server along with any version-managed dependencies. The CDN server uses SPDY to push module dependencies, and automatically minifies the source. This way, any npm module can be loaded efficiently directly in the browser without any install necessary. **Load a single module from a package** ```javascript jspm.import('npm:lodash-node/modern/collections', function(collections) { collections.max([1,2,3,4]); // 4 }); ``` When no version is specified, the latest version is given by the CDN. **Load an AMD module from GitHub** ```javascript jspm.import('github:moment/moment', function() { }); ``` The module format is detected automatically. Additional configuration can be specified in the package.json setting a custom module format, shim config for global scripts, and the main entry point. In this way, true modular configuration is enabled. **Write and load an ES6 module** es6-module.js: ```javascript import { someProperty } from './some-dependency'; export class hello { constructor() { return 'es6'; } } ``` test.html: ```html <script> jspm.import('./es6-module', function(m) { console.log(new m.hello()); }); </script> ``` Traceur is used to parse ES6 in the browser. When moving to production a build can be made with Traceur: package.json: ```javascript { "buildConfig": { "traceur": true, "uglify": true } } ``` ``` jspm build ``` After running the build with the given package.json, ES6 is transpiled into ES5 and AMD. Alternatively upload an ES6 package with the correct package.json to GitHub and the build will be done by the CDN for you. AMD and CommonJS can be loaded in the same way as the above without needing Esprima. **Install a module locally with the CLI** ``` jspm install npm:underscore@1.5 jspm setmode local ``` Installs the version-managed module at `jspm_packages/npm/underscore@1.5.2/`, and auto-generates the configuration file, `config.js`. test.html ```html <script src="jspm_packages/loader.js"></script> <script src="config.js"></script> <script> jspm.import('npm:underscore@1.5', function(_) { _.each([1, 2, 3], console.log); }); </script> ``` All scripts are now loaded from the local system, not the CDN, but requires work exactly the same. #### Next Steps [Join the Google Group](http://groups.google.com/group/jspm-io) to stay informed, engage in discussion, provide feedback or ask for support.
